Desarrollo Web

S1 · ¿Cómo funciona la web?

¿Qué pasa cuando escribes una dirección?

Cuando escribes una dirección como www.wikipedia.org en el navegador y pulsas Intro, ocurre algo parecido a pedir un libro en una biblioteca enorme: tu ordenador pide la página y otro ordenador, muy lejos, se la envía.

Los dos protagonistas: navegador y servidor

Para que veas una página web hacen falta dos ordenadores trabajando juntos:

  • El navegador (Firefox, Chrome…) es el programa de tu ordenador. Pide las páginas y las muestra en pantalla.
  • El servidor web es un ordenador especial que está encendido siempre (24 horas, todos los días) y guarda las páginas. Cuando el navegador se las pide, se las manda.

Esa conversación entre navegador y servidor sigue unas reglas llamadas HTTP (o HTTPS, la versión segura). Por eso las direcciones empiezan por https://.

¿Por qué un servidor y no el ordenador de quien creó la web?

Porque a una página famosa la visitan miles de personas a la vez. Un servidor está preparado para atender muchísimas peticiones sin apagarse. Las webs muy visitadas usan además varios servidores que se reparten el trabajo.

Las partes de una dirección (URL)

Cada página tiene su propia dirección, llamada URL. Fíjate en sus partes con este ejemplo:

Ejemplo: https://es.wikipedia.org/wiki/Internet

  • https → el protocolo: las reglas para pedir la página.
  • es.wikipedia.org → el dominio: el nombre del sitio donde está guardada.
  • /wiki/Internet → la ruta: la página concreta dentro de ese sitio.

Mira una página por dentro

Todas las páginas están escritas con un lenguaje llamado HTML (lo aprenderás en la próxima sesión). Puedes verlo tú mismo:

Inspeccionar una página real

  1. Abre en el navegador una página que te guste (por ejemplo, la Wikipedia).
  2. Haz clic derecho sobre cualquier parte de la página.
  3. Elige «Inspeccionar» (o pulsa la tecla F12).
  4. Se abrirá un panel con el código HTML de la página. Pasa el ratón por encima de las líneas y verás cómo se iluminan las partes de la página.

No te asustes por la cantidad de código: ¡todas las webs se construyen con las mismas piezas que vas a aprender aquí!
